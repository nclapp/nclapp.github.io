Relative, Absolute, and Fixed Positioning

Easy in Theory, Difficult in Practice

I started this week very confident about my HTML skills and enjoyed the refresher, and ended the week with my tail between my legs lamenting CSS and its outrageous determination to completely elude me.

While I understand the usefulness of CSS at its most simple level, i.e. to streamline stylizing websites by allowing you to put a lot of important style info in one place so changing it requires only one change instead of one change in every single element you wish to change, its ability to move, align, and position elements has filled me with frustration and self-doubt. Repeating "I am awesome!" does not work when things aren't working for you!

Part of the problem for me was the interplay between display and position properties. For this blog post, I will try to describe the position property and its multiple behaviors.

Normal flow is simply when each block-level element (like <h1>, <p>, etc.) sits atop the next, so the first one is on top, the second one below, the third one below that, etc. If you have two elements that are only half as wide as the page that could fit on the page side-by-side, they won't; they'll just stack on top of one another even though they could fit. This is normal flow: from top to bottom.

Relative flow is the closest cousin to normal flow, since you are basically adjusting an element's position relative to where it would appear in normal flow. Other surrounding elements are not affected. So if you have three paragraphs, you could move the middle one (for instance) down and to the right any number of pixels, percentage points, or em-spaces. Assign relative flow to an element by inserting the CSS code position: relative in its stylesheet. Then tell it where to move and by how far by adding top, right, left, or bottom offset properties. To move an element down 10 em-spaces, you would write

element {
position: relative;
top: 10em;}

This will move your specified element down 10 em-spaces in relation to where it would normally be, and not affect any of its surrounding elements.

Absolute positioning takes it out of normal flow, and where relative positioning moves an element based on where it would appear in normal flow, absolute positioning moves an element in relation to its parent element. In absolute positioning, other elements act like the one with absolute positioning isn't even there. Much like above in relative positioning, we move an element with absolute positioning using the offset properties. But instead of moving in relation to its normal flow position, it will move in relation to its parent element.

element {
position: absolute;
left: 10em;}

This will move the element 10 em-spaces to the right of where it would ordinarily appear within its containing element, such as a <p> or an <h1>, since we are offsetting its left side by 10em. If there is also another element there 10 em-spaces to the right, they will overlap, since other elements will ignore the position of the absolute positioned item and follow their own specific instructions.
