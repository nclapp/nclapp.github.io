JavaScript and Ruby Compared

Ruby Hashes and JavaScript Object Literals

March 22, 2015

In Ruby, a hash is a great way to store a lot of information that is more complicated than a simple list. While you could keep a list of favorite foods in an array as basically just a list (e.g. ["cheeseburger", "mac and cheese", "lasagna"]), if you want to keep track of other people's favorite foods too, a hash will be far more handy since you can add your friends' names and assign each name favorite foods, instead of having just a long random list of foods.

For example, in Ruby, for yourself alone, the array ["cheeseburger", "mac and cheese", "lasagna"] can help you track your own favorites. But if you want to keep track of your roommates' favorites as well, the format food_hash = {"Me" => ["cheeseburger", "mac and cheese", "lasagna"], "Steve" => ["stale bread", "old sardines", "tofurkey"], "Suzanne" => ["turducken", "meatloaf", "saltines"]} is far more powerful and expressive since you have folded a couple other people's preferences in there as well. Need Suzanne's favorites? Just call the "Suzanne" key from the hash (food_hash["Suzanne"])and it'll return ["turducken", "meatloaf", "saltines"]. Need the second item? Call food_hash["Suzanne"][1], and you'll get ["meatloaf"]. Each person's name is called a "key,"" and the array of their favorite foods is the "value." Very useful.

As we learned this week, JavaScript has much in common with Ruby, but it has some major differences too. One very nice similarity is that JavaScript has an object much like Ruby's hash to help us organize information. Instead of using "hashes" as in Ruby, in JavaScript we use "object literals" to achieve a similar effect.

An example of an object literal in JavaScript is:

var me = {
  name: "Natty",
  age: 34,
  favFoods: ["cheeseburger", "mac and cheese", "lobster"],
  quirk: "loves scuba diving"
}

This is the "me" object I created in this week's Challenge 2. You can see I include some favorite foods as I do in the array example I give above. I also included some other info about me since that was the point of the exercise! To retrieve just my name would simply require the command me.name. Check out the same thing in a Ruby hash: 

me_hash = {
  "name" => "Natty",
  "age" => 34,
  "favFoods" => ["cheeseburger", "mac and cheese", "lobster"],
  "quirk" => "loves scuba diving"
}

To get my name here takes the similar command me_hash["name"]. You can see how similar these are! In JavaScript, we need var to define the object's name, where in Ruby we simply declare it (just like the variables in both languages). And where in Ruby the keys are in quotes, in JavaScript they are properties and don't need to be surrounded by quotes. And instead of the "hash rocket" we know and love from Ruby, we simply use a colon to denote each property value. (JavaScript syntax even more closely resembles the Ruby hash syntax when we use Ruby :symbol notation, but I'm not going to get into that here, check out the RubyDocs for more info.) Now let's go back to my roommates' favorite foods example and look at the two languages next to each other again.

A Ruby hash: 

food_hash = {
  "Me" => ["cheeseburger", "mac and cheese", "lasagna"],
  "Steve" => ["stale bread", "old sardines", "tofurkey"],
  "Suzanne" => ["turducken", "meatloaf", "saltines"]
}

And a JavaScript object literal: 

var roommateFoods = {
  me: ["cheeseburger", "mac and cheese", "lasagna"],
  steve: ["stale bread", "old sardines", "tofurkey"],
  suzanne: ["turducken", "meatloaf", "saltines"]
}

To retrieve Steve's third favorite food in Ruby, we write food_hash["Steve"][2], which returns "tofurkey." In JavaScript, we name the object instead of the hash, and treat Steve's name as a property instead of a Ruby key. So very similarly, we get his third favorite food in JavaScript by writing roommateFoods.steve[2]. So while the syntax is slightly different, the functionality is remarkably similar. It's easy to spot the similarities between food_hash["Steve"][2] and roommateFoods.steve[2]. Very cool!

JavaScript and Ruby have many similarities, and likely as many differences. Lucky for us, the functionality delivered by hashes in Ruby is available to us in JavaScript as well through the use of object literals. Good thing! Thanks for reading!
